Airways

CHALLENGE

My capstone project for Galvanize's Web Development Immersive. All students were tasked with building a full-CRUD website by themselves using the React framework. As had become a tradition at this point I wanted to use multiple third party API calls alongside developing an API for my own database.

PROJECT

Airways is a web application for asthmatics and people with allergies, allowing them to track air quality and pollen data for the places they visit throughout their day. This project required me to make multiple asynchonous calls, first to Google Maps to translate a location's address into map coordinates, then to an API called Breeze-O-Meter to fetch air data for those coordinates, before finally displaying the results to the user. User's could also sign up for email alerts via MailGun to recieve a saved location or routes air data every morning.

TECHNOLOGIES USED

React/Redux * ReactStrap * Javascript ES6 * HTML5/CSS3 * Bootstrap * Node * Heroku * Express * Axios * Knex * Google Maps * Breeze-O-Meter * MailGun

The Archives

CHALLENGE

This was our Galvanize Q3 final project, a 2-person team effort to create a full-CRUD website utilizing the new utilizing React for the front-end. My partner and I wanted to push ourselvesand not only utilize a third-party API but also create the front-end using Redux.

PROJECT

The Archives is a site for players of Magic The Gathering that let's them create decks and analyze their composition. We used a third-party API to allow users to search a massive database of cards, view their information, and then add them to a specific deck. Chris Says

CHALLENGE

This application was completed for my Galvanize Q1 Project in which I was tasked with demonstrating my front-end skills. It had to be a fully-interactive site that used Javascript to take user input, store something in local memory, and return something. I decided I wanted to make a game my 4-year-old son could play, meaning it had to be something simple with minimal reading.

PROJECT

Chris Says is a pattern matching game modeled after the old electronic game, Simon Says, from the 1980s. It has multiple difficulties and once selected runs a recursive function that flashes a sequence of red circles across the play field. Players must recreate the pattern which increases in length with each successful sequence until the either complete a full 16-circle pattern or make a mistake.

TECHNOLOGIES USED

Javascript ES6 * HTML5/CSS3 * Bootstrap * Node

Roolan.com

CHALLENGE
